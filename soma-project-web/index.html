soma-next in the browser — phase 1d

A WebAssembly build of soma-next loaded in this tab boots a real Runtime with the full DefaultPortRuntime pipeline — lifecycle gate, sandbox check, policy check, auth check, input schema validation. The three in-tab ports (dom, audio, voice) are registered on top of the bootstrap flow the same way native dylib ports are registered on a server soma-next. soma_invoke_port dispatches through DefaultPortRuntime::invoke, exactly the same code path every native proof project uses. Everything above this paragraph is static HTML — nothing below it is.
